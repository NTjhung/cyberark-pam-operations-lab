# CyberArk PAM Operations Lab

## Project Overview

This project demonstrates CyberArk Privileged Access Management operations based on hands-on CyberArk access. The lab focuses on privileged account management, safe design, access request workflows, password rotation planning, session monitoring concepts, CPM/PSM troubleshooting concepts, break-glass handling, and audit evidence documentation.

Screenshots are not included because screenshot capture was not permitted in the CyberArk environment. Instead, this project uses original runbooks, workflow documentation, sample data, diagrams, and evidence summaries to document the work in a safe and non-confidential way.

## Business Problem

Organizations need to control, monitor, and audit privileged access across systems, applications, infrastructure, and administrative accounts. Without a Privileged Access Management process, privileged credentials may be shared, unmanaged, overused, or left active longer than necessary.

CyberArk helps organizations reduce privileged access risk by supporting password vaulting, access control, password rotation, session isolation, session monitoring, privileged account onboarding, and audit evidence.

## Key PAM Concepts Demonstrated

### Privileged Access Management

Privileged Access Management helps control, monitor, and secure accounts with elevated access.

### Safes

Safes are logical containers used to organize, control, and secure privileged accounts.

### Privileged Account Onboarding

Privileged account onboarding is the process of adding privileged accounts into PAM control so they can be managed, monitored, and audited.

### Password Rotation

Password rotation helps reduce credential risk by changing privileged account passwords based on policy or after use.

### CPM

Central Policy Manager is commonly associated with password management, password verification, password changes, and password reconciliation workflows.

### PSM

Privileged Session Manager is commonly associated with privileged session isolation, monitoring, recording, and controlled access workflows.

### Break-Glass Access

Break-glass accounts provide emergency access when normal privileged access workflows are unavailable.

### Audit Evidence

Audit evidence includes access requests, approvals, onboarding records, password rotation records, session records, troubleshooting notes, and remediation notes.
