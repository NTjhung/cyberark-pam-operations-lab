# Screenshot Restriction Note

## Purpose

This document explains why screenshots are not included in the CyberArk PAM Operations Lab.

## Screenshot Restriction

Hands-on CyberArk access was available for this project. However, screenshot capture was not permitted in the CyberArk environment.

To respect the access restrictions, this project does not include screenshots from the CyberArk platform.

## Documentation Approach

Instead of screenshots, this project uses original and sanitized artifacts, including:

- PAM overview documentation
- Safe design matrix
- Privileged account onboarding workflow
- Access request workflow
- Password rotation plan
- PSM session monitoring workflow
- CPM troubleshooting checklist
- PSM troubleshooting checklist
- Break-glass access plan
- Audit evidence packet
- Sample privileged account inventory
- Sample safe design matrix
- Sample password rotation schedule
- PAM workflow diagram
- Lab evidence summary

## Why This Matters

CyberArk environments often contain sensitive privileged access information, including account names, safe names, system details, administrative workflows, audit records, and session data.

Avoiding screenshots helps protect:

- Privileged account details
- System and infrastructure information
- Safe names and access structure
- User and administrator information
- Session monitoring information
- Audit and security records
- Confidential environment details
