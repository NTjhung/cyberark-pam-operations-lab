# CyberArk PAM Lab Evidence Summary

## Purpose

This document summarizes the evidence created for the CyberArk PAM Operations Lab.

This project is based on hands-on CyberArk access. Screenshots are not included because screenshot capture was not permitted in the CyberArk environment. Evidence is provided through original documentation, sample data, workflow diagrams, and sanitized operational notes.

## Hands-On Access Statement

Full CyberArk access was available for this lab. The project was documented without screenshots because screenshot capture was not permitted in the CyberArk environment.

The documentation focuses on non-confidential operational understanding, including safe design, privileged account onboarding, password rotation, access request approvals, session monitoring, troubleshooting, break-glass handling, and audit evidence.

## PAM Areas Covered

This project covers the following PAM areas:

- Privileged account management
- CyberArk safe design
- Access request workflow
- Approval workflow
- Password rotation planning
- PSM session monitoring concepts
- CPM troubleshooting concepts
- PSM troubleshooting concepts
- Break-glass handling
- Audit evidence documentation
- Privileged access review planning
- Least privilege concepts

## Example Audit Questions Answered

This lab provides documentation that helps answer common PAM audit questions:

1. Are privileged accounts inventoried?
2. Are privileged accounts assigned to safes?
3. Are safe owners documented?
4. Are approval requirements documented?
5. Are password rotation requirements documented?
6. Are privileged sessions monitored or reviewed?
7. Are break-glass accounts handled separately?
8. Are troubleshooting issues documented?
9. Is audit evidence retained?
10. Are access reviews part of the governance process?

## Evidence Limitations

This evidence packet does not include screenshots, exports, or confidential data from the CyberArk environment.

The following items were intentionally excluded:

- Screenshots from CyberArk
- Real privileged account names
- Real safe names
- Real user information
- Real system names
- Real session records
- Real audit logs
- Real customer or organization data
